# NIST CSF 2.0 Mini Compliance Program

## 📖 Project Overview
This project demonstrates a **Governance, Risk, and Compliance (GRC)**-style cybersecurity assessment using the **NIST Cybersecurity Framework (CSF) 2.0**.

The assessment evaluates how an organizations, identifies control gaps, documents risks, and provides a practical remediation roadmap aligned with real-world cybersecurity practices.

This project is designed as a **portfolio artifact** to showcase skills in:
- Cybersecurity risk assessment
- Control gap analysis
- Risk register development
- Compliance framework application
- Executive-level reporting

---

## 🎯 Objective
The objective of this project is to:

- Assess cybersecurity maturity using NIST CSF 2.0
- Identify missing or partially implemented controls
- Document risks in a structured format
- Provide actionable remediation recommendations
- Simulate real-world GRC assessment and reporting

---

## 🏢 Company Scenario
Scaler Enterprises. a fortune 800 company based in North America.

### Environment
- Microsoft 365 (email & file sharing)
- Company-issued laptops
- Shared cloud storage
- Office Wi-Fi network
- Basic antivirus protection
- Outsourced IT support

### Data Handled
- Client Personally Identifiable Information (PII)
- Confidential legal documents
- Employee records
- Financial and billing data
- Internal communications

---

## 🧩 Framework Used
This project is based on the **NIST Cybersecurity Framework (CSF) 2.0**, covering:

- Govern (GV)
- Identify (ID)
- Protect (PR)
- Detect (DE)
- Respond (RS)
- Recover (RC)

---

## 🛠️ Methodology
The assessment follows a simplified GRC approach:

1. Define business and technical environment
2. Map controls to NIST CSF 2.0
3. Evaluate implementation status
4. Identify control gaps
5. Develop risk statements
6. Assign qualitative risk ratings
7. Recommend remediation actions

---

## 📊 Key Findings

| Area | Issue | Risk Level |
|-----|------|-----------|
| Asset Management | No formal asset inventory | High |
| Access Control | MFA not fully enforced | High |
| Incident Response | No documented IR plan | High |
| Monitoring | Limited logging/alerting | Medium |
| Security Awareness | No formal training program | Medium |
| Backup & Recovery | Backups not tested | High |

---

## ⚠️ Example Risk Statement
> If user access rights are not regularly reviewed and aligned with job responsibilities, former employees or unauthorized users may retain access to sensitive systems, increasing the risk of data exposure or misuse.
---

## 🛣️ Remediation Roadmap

### 🔴 Immediate (0–30 Days)
- Enforce MFA across all systems
- Create asset inventory
- Define incident response contacts
- Review admin access

### 🟠 Near-Term (30–90 Days)
- Develop core security policies
- Launch security awareness training
- Implement access reviews
- Define backup procedures

### 🟢 Medium-Term (90–180 Days)
- Formalize vendor oversight
- Implement centralized logging
- Conduct incident response testing
- Test backup recovery
